业务流程总览

从页面操作到数据库读写的主要路径。

1. 加载流程选项只返回当前用户事项中实际出现的有效流程。
2. 事项查询按用户、流程、证券、日期和状态分页。
3. 选择代表步骤待办取本人最新 pending；其他取每申请最新步骤。
4. 识别场景根据池类型和主体类型判断跳转页面。
5. 进入处理/查看待办进入审批，已结束进入只读详情。

事项分页与可见范围

普通用户只能看到曾由自己处理或待处理的事项，管理员可穿透查看。

POST /api/v1/myMatters/queryMyMattersPage

待办代表步骤

  • stepStatus=pending 时按 adjust_log_id 分组取最大 pending 步骤ID。
  • 普通用户同时要求 handler_id=currentUserId；未传用户时强制无结果。
  • 管理员 1 不加处理人限制。

已办与筛选

  • 非待办取每条申请的最新步骤。
  • 普通用户还要求该申请至少存在一条本人处理步骤。
  • 支持流程、证券、开始日期、描述、审核状态、发起人；completed 要求同批次不存在 pending。

流程选项与业务场景

列表组合申请、步骤、流程和证券大类生成跳转信息。

POST /api/v1/myMatters/queryFlowOptionList

流程下拉

  • 从当前用户可见事项的最新步骤反查 wf_flow_node 和有效流程定义。
  • 按流程名称、ID排序并去重。

场景判断

  • pool_type='crmw' → CRMW 调整。
  • company 类型且目标池ID在 15/16/17 → 主体禁投调整。
  • 其他 → 证券池调整；待办与非待办分别进入处理或查看模式。

涉及表、用途与触发场景

读写口径包含页面初始化、主业务接口及其正常后续流程。

表名 用途 读写 触发场景
ip_adjust_log 事项对应的调库申请 查询 业务对象、状态、批次和发起人
ip_adjust_step 待办/已办步骤及当前处理人 查询 可见范围、代表步骤、完成判断
dict_security_type 识别 company 等证券大类 查询 业务场景判断
wf_flow_node 步骤关联的流程节点 查询 反查流程定义
wf_flow_definition 流程名称、Key和描述 查询 列表显示和流程下拉
ip_investment_pool 目标池完整路径 查询 替换流程描述中的叶子池名称

关键规则与当前实现说明

  • 管理员用户ID在当前代码中固定为 1，待办不加 handler 限制。
  • completed 判断是同批次无 pending 步骤，不只看当前步骤一行。
  • 业务场景：pool_type=crmw → CRMW；company 且目标池 15/16/17 → 禁投主体；否则证券池（依赖种子池 ID）。
  • 待办跳转审核页，已结束跳转详情 view；本页只读，写操作在对应 submitAdjustAudit。
  • 审核状态等 code 由前端字典映射展示。
