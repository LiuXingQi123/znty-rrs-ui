业务流程总览

从页面操作到数据库读写的主要路径。

1. 流程列表按关键字、状态和分类分页查询。
2. 新建草稿新增流程定义和 verNum=1 草稿版本。
3. 画布编辑编辑节点、连线、处理人、任务和条件。
4. 保存/发布保存草稿或校验后发布 active 版本。
5. 运行消费调库运行时读取归一化节点、连线和审批配置。

定义与版本管理

流程定义控制名称、Key、分类和状态；版本保存完整画布快照。

POST /api/v1/flows/queryFlowPage · addFlow · editFlowDraft · editFlowToPublished

定义与草稿

  • flowKey 必填且全局唯一；新增同时创建初始 draft 版本。
  • 详情优先返回已有草稿，否则返回最新版本作为编辑基础。
  • active 流程首次保存新草稿时，定义状态回退为 draft。

发布与状态

  • 发布前校验节点、唯一开始节点、结束节点、连线及配置。
  • 发布把草稿版本转 active，并更新流程定义状态。
  • 停用仅允许 active→disabled；删除为定义表软删除。

画布归一化与字典

版本中的 canvas_nodes/canvas_edges 是画布快照；归一化表供运行时遍历。

POST /api/v1/flows/queryFlowVersionList · queryFlowVersionDetail · dict/*

全量同步

  • 每次保存/发布按依赖顺序删除该版本旧归一化数据。
  • 重新插入节点、审批配置/处理人、自动任务、通知、条件节点、连线和条件规则。
  • 删除旧行和新增新行都记录对应 evt 事件。

人员和字典

  • 角色、用户来自 ais_inv_analysis 系统表。
  • 自动任务和条件字段来自后端静态字典；角色字典表仍提供查询。
  • 运行时主要读取定义、版本、节点、连线、条件规则和审批配置。

关键规则与当前实现说明

  • 画布 JSON 是设计器快照真源，归一化表是运行时派生数据；两套数据必须同步。
  • 发布前会阻止无节点、开始/结束节点异常和关键配置缺失。
  • 删除流程当前未实现业务引用检查，直接把定义 is_deleted 置为1。
  • 停用流程中的“检查运行中实例”仍是 TODO，且没有 disabled→active 的逆向接口。
  • 运行时目前没有统一流程实例表，具体业务用自己的日志/步骤表承载实例。
