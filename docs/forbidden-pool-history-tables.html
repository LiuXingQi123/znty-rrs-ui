禁投池历史

从调库日志中查询目标池类型为 forbidden / observe / blacklist 的历史调整，覆盖主体与证券、调入调出、审核状态和操作人员，并按可查看权限过滤。

5 项表/表组1 个主要接口依据当前代码实现
流程总览禁投池历史查询涉及表规则说明文档目录

业务流程总览

从页面操作到数据库读写的主要路径。

1. 权限范围按 currentUserId 解析 viewable 可查看池。
2. 输入条件主体、证券、日期、调整人、方向及状态。
3. 限定范围目标池 pool_type ∈ 三类风险池。
4. 分页排序submit_time / adjust_batch_no / id 倒序。
5. 路径回填Service 将叶子池名替换为完整路径。

禁投池历史查询

历史来源为调库申请日志，包含已结束和进行中记录。

POST /api/v1/forbiddenPoolHistory/queryForbiddenPoolHistoryPage

查询主线

  • 以 ip_adjust_log 为主表，过滤 is_deleted=0。
  • 内连接投资池：pool_type IN ('forbidden','observe','blacklist')。
  • 注入 viewablePoolIds（VIEWABLE）；空集合无数据。
  • 左连字典、证券主数据、Wind 主体表（company 行取发行人名）。
  • 排序与证券/主体/CRMW 历史统一：submit_time DESC, adjust_batch_no DESC, id DESC。

筛选口径

  • 主体代码：Wind 主体码 或 债券 issuer_code 模糊。
  • 主体名称：COALESCE(wind 主体名, 债券 issuer) 模糊。
  • 证券代码/简称匹配日志字段；另支持提交日期、调整人、方向、审核状态。

涉及表、用途与触发场景

读写口径包含页面初始化、主业务接口及其正常后续流程。

表名 用途 读写 触发场景

关键规则与当前实现说明

  • 页面只读，不修改调库记录。
  • 以目标池类型过滤：主体级手工申请、联动/互斥、以及旗下债券自动同步日志都可能出现。
  • 同批多行仍分条展示；靠统一 submit_time + 批次号排序聚拢，不是合并为一行。
  • 审核状态展示原始 code，中文由前端字典映射。
