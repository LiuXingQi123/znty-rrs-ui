业务流程总览

从页面操作到数据库读写的主要路径。

1. 查询列表按临时代码、名称分页并展示临时及正式信息。
2. 加载选项查询发行主体和证券类型；转正时远程搜正式证券。
3. 新增临时证券写主表 + 同步 rrs_securityinfo 占位（security_source=temporary）。
4. 转正式选正式主数据；在途改字段、已在池出/入；临时占位置 D。
5. 取消/删除取消仅 temporary；删除仅 temporary 且无核心引用。

查询与下拉选项

列表与详情的删除过滤口径存在差异。

POST /api/v1/tempSecurityCode/queryTempSecurityCodePage · queryTempSecurityCodeOptions · queryFormalSecurityOptionList

列表查询

  • 主表为 rrs_temp_security_code，关联两次证券类型字典。
  • 支持临时代码、临时名称模糊查询；排序对齐老系统状态优先。
  • 当前列表未过滤 is_deleted，已删除记录仍可显示；详情按 ID 过滤已删除。

选项与正式选券

  • 发行主体来自 ais_inv_ods.wind_cbondissuer（远程搜索，最多 50 条）。
  • 证券类型来自未删除且 category_type!='company' 的字典。
  • 转正正式证券：queryFormalSecurityOptionList 搜 rrs_securityinfo（排除 temporary 占位与退市，最多 50 条）。

新增、转正式、取消与删除

所有变更操作均在事务中执行；转正式对齐老系统 refreshTempSecurity 分叉。

POST /api/v1/tempSecurityCode/addTempSecurityCode · editTempSecurityCodeToUpdated · editTempSecurityCodeToCancelled · deleteTempSecurityCode

新增

  • 状态固定 temporary，仅写临时字段。
  • 同步插入 rrs_securityinfo（wind_code=临时代码，security_source=temporary）。
  • 若临时码已在主数据存在则拒绝新增。

转正式（核心）

  • 仅 temporary；正式码从主数据选券，临时信息只读；正式码不能等于临时码。
  • 在途（ip_adjust_log audit_status=00/11）：只改日志上证券/CRMW 字段为正式码，不写出/入业务单。
  • 已在池（ip_pool_status 等 audit_status=20）：每池写「债券临时代码调出」+ 软删临时状态；正式未在池再写正式调入 + 新建池状态。
  • 正式主数据已存在则不再 upsert；临时占位置 security_status=D。

取消和删除

  • 取消发行仅 temporary：主数据到期/止息改为前一日并置 D，状态 cancelled。
  • 删除仅 temporary，且无调库/池引用；status=deleted、is_deleted=1。
  • updated / cancelled 不可再删除（对齐老系统）。

转正时调库日志条数（对齐老系统）

以「临时已在池 → 转正式」且正式尚未在同池为例。

完整时间线常见 3 条

  • ① 历史：当初临时代码调入时产生的调入日志（转正不改不删）。
  • ② 转正当下：临时代码调出（reason=债券临时代码调出，audit_status=20）。
  • ③ 转正当下：正式代码调入（继承原原因/调整人，audit_status=20）。

分支差异

  • 正式码已在同池：转正只新增出库 1 条，不写正式入库。
  • 仅在途未在池：不新增出/入业务单，只改在途日志字段。
  • 池/日志上 CRMW 字段指向临时码：仅字段替换（对齐老 findInvestPoolStatusLC）。

关键规则与当前实现说明

  • 状态流转：temporary → updated / cancelled；删除仅 temporary → deleted。updated/cancelled 为冻结态，不可再转正式/取消/删除。
  • 转正式必须从主数据选已有正式券，不再对正式 wind_code 做 upsert 覆盖；正式码不能等于临时码。
  • 转正分叉：在途（audit_status=00/11）只改字段；已在池（20）写出库+（正式未在同池时）入库；终态历史日志不改码，保留审计轨迹。
  • O32 识别：按 temp_security_code + status=temporary 判断，临时代码转人工 pending。
  • 调入信用债大库：临时占位无主体内评时默认 grade_code=4（对齐老系统 lon 默认 8.0 最低档语义）。
  • 列表未过滤软删除而详情过滤软删除，是当前实现的明确不一致点；已删除行仍可在列表看到「已删除」标签。
  • 临时主体代码不变时保留名称快照，即使主体主数据名称后来变化也不会自动刷新。
  • 前端投资市场为统一 8 码（含 COMPANY=主体等）；发行主体远程搜索 wind，最多 50 条。
