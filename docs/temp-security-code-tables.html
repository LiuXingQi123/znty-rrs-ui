业务流程总览

从页面操作到数据库读写的主要路径。

1. 查询列表按临时代码、名称分页并展示临时及正式信息。
2. 加载选项查询发行主体和证券类型。
3. 新增临时证券校验主体、市场、类型、日期和代码唯一性。
4. 转正式/取消临时状态可更新为正式证券或取消发行。
5. 删除任意未删除状态均可软删除。

查询与下拉选项

列表与详情的删除过滤口径存在差异。

POST /api/v1/tempSecurityCode/queryTempSecurityCodePage · queryTempSecurityCodeOptions

列表查询

  • 主表为 rrs_temp_security_code，关联两次证券类型字典。
  • 支持临时代码、临时名称模糊查询，按创建时间和ID倒序。
  • 当前列表未过滤 is_deleted，因此已删除记录仍可显示；详情按ID会过滤已删除。

选项数据

  • 发行主体来自 ais_inv_analysis.t_inv_company。
  • 证券类型来自未删除且 category_type!='company' 的字典。

新增、转正式、取消与删除

所有变更操作均在事务中执行。

POST /api/v1/tempSecurityCode/addTempSecurityCode · editTempSecurityCodeToUpdated · editTempSecurityCodeToCancelled · deleteTempSecurityCode

新增和转正式

  • 新增状态固定 temporary，仅写临时字段。
  • 转正式仅允许 temporary；先按正式 securityCode 对 rrs_securityinfo 做 INSERT/UPDATE，再更新临时代码状态为 updated。
  • 正式证券 issuer 使用主体名称快照。

取消和删除

  • 取消发行仅允许 temporary，状态改为 cancelled，不写正式证券。
  • 删除只要求记录存在且未删除，temporary/updated/cancelled 均可删。
  • 删除更新 status=deleted、operation_type=delete、is_deleted=1。

涉及表、用途与触发场景

读写口径包含页面初始化、主业务接口及其正常后续流程。

表名 用途 读写 触发场景
rrs_temp_security_code 临时与正式代码映射及状态主表 查询 新增 更新 列表、新增、转正式、取消、软删除
rrs_securityinfo 正式证券基础信息 查询 新增 更新 临时代码更新为正式证券时按 wind_code upsert
dict_security_type 临时/正式证券类型字典与合法性校验 查询 列表类型名称、下拉和提交校验
ais_inv_analysis.t_inv_company 发行主体选项与主体名称 查询 新增或变更关联主体

关键规则与当前实现说明

  • 状态流转：temporary → updated / cancelled / deleted；updated、cancelled 只能继续删除。
  • 正式证券 upsert 以 wind_code=securityCode 判重，后更新的同代码记录可能覆盖既有主数据。
  • 列表未过滤软删除而详情过滤软删除，是当前实现的明确不一致点。
  • 临时主体ID不变时保留名称快照，即使主体主数据名称后来变化也不会自动刷新。
