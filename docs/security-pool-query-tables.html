业务流程总览

从页面操作到数据库读写的主要路径。

1. 初始化筛选加载投资池树、证券类型和页面状态选项。
2. 组合查询按池、证券、日期、调整人、发行人等条件分页。
3. 状态计算根据到期日计算存续或到期。
4. 路径回填将叶子池名称转换为完整投资池路径。
5. 收藏维护按用户新增或逻辑取消我的证券收藏。

证券池分页查询

只展示已经生效且未删除的证券池状态。

POST /api/v1/securityPoolQuery/querySecurityPoolPage

查询主线

  • 主表 ip_pool_status 固定过滤 audit_status='20'、is_deleted=0。
  • 关联投资池、证券信息和证券类型字典。
  • 按入池时间、状态ID倒序分页，Service 回填池全路径。

筛选条件

  • 支持多池、证券代码/简称/类型、入池日期、调整人和发行人。
  • 存续/到期由 maturity_date 与当前日期比较。
  • “我的证券”通过当前用户的有效 my_security_pool 记录过滤。

我的证券池收藏

收藏仅影响个人视图，不改变正式证券池状态。

POST /api/v1/securityPoolQuery/addSecurityToMyPool · deleteSecurityFromMyPool

新增收藏

  • userId 和 securityCode 必填。
  • 若已存在有效收藏则直接返回；否则新增 status='use' 记录。

取消与回显

  • 取消时按用户+证券代码更新收藏状态。
  • 列表初始化可一次查询当前用户收藏证券代码集合。

涉及表、用途与触发场景

读写口径包含页面初始化、主业务接口及其正常后续流程。

表名 用途 读写 触发场景
ip_pool_status 已生效证券池状态，是列表主数据 查询 分页、证券类型选项
ip_investment_pool 目标池名称与完整层级路径 查询 列表关联、路径回填、池树筛选
rrs_securityinfo 发行人、全称、票息和日期等证券资料 查询 列表字段和存续状态判断
dict_security_type 证券类型名称 查询 列表展示和类型下拉
my_security_pool 当前用户证券收藏 查询 新增 更新 收藏、取消收藏、仅看我的证券

关键规则与当前实现说明

  • 正式池查询只认 ip_pool_status.audit_status='20'。
  • 证券状态不是独立表字段，当前查询通过证券到期日动态计算。
  • 收藏表与正式证券池状态完全分离，取消收藏不会调出证券。
  • 前端状态下拉为本地字典；后端也提供 active/matured 选项接口。
