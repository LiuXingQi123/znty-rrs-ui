业务流程总览

从页面操作到数据库读写的主要路径。

1. 加载池树common/queryPoolTreeList 作多选筛选。
2. 权限交集VIEWABLE 可查看池 ∩ 页面勾选 poolIds。
3. 主体过滤仅保留 category_type='company' 的状态。
4. 有效状态只认 audit_status='20' 且未删除。
5. 路径展示Service 回填完整投资池路径。

主体池分页查询

直接查主体在池状态，不聚合旗下债券；与「禁投池查询」互补（后者含三类风险池内主体+证券）。

POST /api/v1/companyPoolQuery/queryCompanyPoolPage

查询主线

  • 以 ip_pool_status 为主表。
  • 内连接 dict_security_type，固定 category_type='company'。
  • 固定 audit_status='20'、is_deleted=0；排序 entry_time DESC, id DESC。
  • Service：可查看池 ID 与页面 poolIds 求交集后写入 viewablePoolIds。

筛选和路径

  • 支持多目标池、主体代码（匹配 security_code）、入池起止日期、调整人。
  • 关联投资池取叶子名，再回填全路径。
  • 当前 Mapper 无主体名称/评级等扩展筛选（名称以状态快照字段为准）。

涉及表、用途与触发场景

读写口径包含页面初始化、主业务接口及其正常后续流程。

表名 用途 读写 触发场景
ip_pool_status 已生效主体池状态主数据 查询 主体池分页和筛选
dict_security_type 识别 company 类型 查询 限定主体记录
ip_investment_pool 目标池名称和完整路径 查询 列表关联与路径回填
ip_pool_permission 等 解析用户可查看池（VIEWABLE） 查询 viewablePoolIds 与页面池多选交集

关键规则与当前实现说明

  • 页面只读，不修改主体池状态；不发起调库（调库走「禁投池调整」）。
  • 主体身份由 security_type → 字典 category_type='company' 判断，不限定 pool_type 必须为 forbidden。
  • 列表字段主要来自池状态快照；未强制关联 rrs_securityinfo / wind_cbondissuer。
  • 无权限可见池时结果为空（viewable 空集合）。
