查询、校验与操作逻辑

按当前前后端实现归纳主要触发场景。

主查询口径

查询以 ip_pool_status_crmw 为唯一状态事实表，不读取普通证券池的 ip_pool_status 作为替代。过滤条件按代码、简称、标的证券及目标池组合生成。

有效状态

只展示未逻辑删除的 CRMW 池状态；投资池本身也必须未删除。这样历史调出记录不会继续出现在当前池查询中。

池信息回填

通过 target_pool_id 关联投资池，返回池名称、类型及路径所需字段；Service 可进一步回填完整树路径。

只读页面

该页面不创建申请、不写步骤，也不改变 CRMW 状态。所有调入调出必须从 CRMW池调整或审批页面完成。

涉及表与读写场景

“条件写入”表示仅在特定流程、附件或审批分支中发生。

关键规则与边界

帮助区分当前状态、历史记录和流程实例。

与证券池查询的区别

普通证券池查询读取 ip_pool_status；CRMW池查询读取独立的 ip_pool_status_crmw。

查询不等于历史

逻辑删除的调出状态不在此页展示，完整操作轨迹应查看 CRMW池调整历史。

无写操作

筛选、分页和排序均为只读，不会触发流程或状态变更。
