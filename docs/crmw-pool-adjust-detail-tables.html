从页面操作到数据库读写的主要路径。

1. 识别模式

view 只读查看；adjust 允许首次提交调库申请。

2. 并行加载

读取CRMW 凭证与标的证券详情、当前池、可调池、日志和步骤。

3. 选择方向

仅叶子池可选，当前在池决定可调入或可调出范围。

4. 后端校验

检查到期、进行中流程、容量、来源、联动和互斥。

5. 首次提交

写申请、附件及必要步骤；详情页不处理后续审批动作。

查询、校验与操作逻辑

按当前前后端实现归纳主要触发场景。

查看模式

按业务代码与批次加载日志、步骤、附件；只读展示调整原因/建议（从日志回填），不触发 check 或写表。

调整模式

加载有权限的池树并根据当前有效在池状态二次过滤；用户选择叶子池、方向和流程后调用后端校验。

校验返回

手工+联动/互斥/关联项一并返回；failReasons 与 warnings 分列；失败时调整说明为类型方向前缀+原因（申请主页才仅展示原因）；同组底色区分。仅通过手工项可选流程。

首次提交

请求携带 crmwScode/crmwName/crmwStype 与 securityCode（无 mktcode）；直通锁池复核后只改 ip_pool_status_crmw。

职责边界

审批中的同意、驳回、会签推进及修改后重提属于对应“审核”页面，本详情页不调用 FlowController。

附件处理

无附件走 JSON 提交，有附件走 multipart；sys_attachment 通过 adjustLogId 关联申请。

关键规则与边界

帮助区分当前状态、历史记录和流程实例。

模式边界

view 模式始终只读；adjust 模式仅负责首次申请。

当前状态口径

当前池必须读取最终状态表，申请日志只代表过程和历史。

重复提交

存在待处理（pending）步骤或目标池状态已变化时，后端重新校验并阻止过期提交。

文档口径来自当前页面、Controller、Service、Mapper XML 与对应 requirements；后续代码调整时应同步维护本页。
