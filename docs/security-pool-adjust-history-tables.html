基于调库申请日志查询证券池调入、调出及审批结果，关联证券、投资池和类型字典形成审计视图。

业务流程总览

从页面操作到数据库读写的主要路径。

1. 加载筛选加载投资池树和历史中出现的证券类型。
2. 组合过滤按证券、池、方向、状态、日期、人员等查询。
3. 关联补充关联证券发行人、到期状态及目标池名称。
4. 路径回填将目标池叶子名称替换为完整路径。
5. 历史展示按批次、提交时间和日志ID倒序展示。

历史分页查询

历史来源是申请日志，不是当前池状态。

POST /api/v1/adjustHistory/queryAdjustHistoryPage

主查询

  • 以 ip_adjust_log 为主表，固定过滤 is_deleted=0。
  • 关联投资池、证券基础信息和证券类型字典。
  • 按批次号、提交时间、日志ID倒序分页。

筛选与派生

  • 支持池、证券代码/简称/类型、方向、审核状态、提交日期、调整人、发行人。
  • 证券存续/到期通过 rrs_securityinfo.maturity_date 动态判断。
  • “我的债券”实际按 adjuster_id=currentUserId 过滤本人发起记录。

涉及表、用途与触发场景

读写口径包含页面初始化、主业务接口及其正常后续流程。

表名 用途 读写 触发场景
ip_adjust_log 证券池调整申请和审批历史主表 查询 历史分页、方向和审核状态筛选
ip_investment_pool 目标池名称及路径 查询 列表关联、完整路径回填
rrs_securityinfo 发行人和到期日 查询 发行人筛选、证券状态计算
dict_security_type 证券类型名称 查询 列表展示和类型选项

关键规则与当前实现说明

  • 页面为只读查询，不修改任何业务表。
  • 历史记录包括手工、联动、互斥和自动调整，具体范围由筛选条件决定。
  • “我的债券”命名容易误解，当前代码口径是本人作为调整人发起的日志。
