证券池批量调整

批量选择启用叶子投资池和候选证券，逐券复用单证券调库校验，按证券分组创建调库申请并进入统一审批流程。

业务流程总览

从页面操作到数据库读写的主要路径。

1. 选择目标池按用户可调整权限查询启用叶子池并回填当前数量。
2. 筛选证券调入排除已在池证券，调出仅保留已在池证券。
3. 批量校验逐证券执行到期、在池、容量、关系及流程匹配。
4. 批量提交按证券分组写申请、步骤和附件，共享批量上下文。
5. 审批生效统一审批通过后更新证券池状态并按条件沉淀报告。

目标池与候选证券查询

先选有权限的启用叶子池，再按方向查询候选证券。

POST /api/v1/batchSecurityPoolAdjust/queryPoolPage · querySecurityPage

目标池查询

  • 管理员可查看全部；普通用户依据 ip_pool_permission 的 adjustable 权限过滤。
  • 只返回 status='enabled'、未删除且没有启用子节点的叶子池。
  • 汇总 ip_pool_status 的有效证券数量并回填完整池路径。

候选证券查询

  • 目标池必须存在、启用、为叶子池且当前用户有调整权限。
  • 调入使用 NOT EXISTS 排除目标池有效状态；调出使用 EXISTS 仅保留目标池有效状态。
  • 支持证券代码、简称和市场过滤，按证券主键倒序分页。

批量校验与提交

批量层负责聚合，单券规则与证券池调库保持一致。

POST /api/v1/batchSecurityPoolAdjust/checkAdjust · addAdjustLogWithFiles

逐券校验

  • 每个证券转换为单证券 AdjustCheckReq。
  • 自动扩展 in_linked、out_linked 和 in_mutex 调出项，并用池ID+方向去重。
  • 仅校验通过的手工项选择流程，联动/互斥项继承同组手工流程。

提交和审批

  • 按 securityCode 分组，逐组创建 ip_adjust_log。
  • 直通流程立即写入或删除 ip_pool_status；非直通创建 ip_adjust_step。
  • 同一次批量提交共享附件上下文；后续由证券池调库审批服务推进。

关键规则与当前实现说明

  • 批量不是一条总申请：后端按证券分组，再为手工、联动、互斥明细分别保存调库日志。
  • 方向只允许 in 或 out；手工明细方向必须与本次批量方向一致。
  • 手工项必须指定审批流程；联动和互斥项复用同组手工项流程。
  • 审批运行、会签/抢占、驳回和最终落库复用证券池调库流程逻辑。
