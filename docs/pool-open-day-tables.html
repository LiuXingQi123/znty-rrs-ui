RRS 配置

投资池开放日维护

维护 ip_pool_open_day 开放区间；池级开关 open_day_adjust=1 时参与调库校验。

主表 ip_pool_open_day

列说明
id主键
pool_id投资池 ID → ip_investment_pool.id
begin_date / end_date开放区间（含边界）
description描述，最长 200
is_deleted0 正常 / 1 已删除
crte_time / updt_time创建 / 修改时间

接口

POST /api/v1/poolOpenDay/* · 页面 pages/pool_open_day.html

  • queryPoolOpenDayPage — 分页（池、日期区间、描述），返回 poolFullName / openDayAdjust
  • addPoolOpenDay — 新增（有子节点的目录不可配置；起止日校验）
  • editPoolOpenDay — 修改
  • deletePoolOpenDay — 逻辑删除 is_deleted=1
  • 排序：begin_date DESC, id DESC；列表「区间状态」前端按今天算：未开始 / 开放中 / 已结束
  • 投资池选项：树选择展示全称，不展示池编码

调库关联

证券池 / 禁投 / CRMW / 批量：目标池 open_day_adjust=1 时，调入与调出均校验当日是否落在开放区间：

begin_date <= today AND end_date >= today AND is_deleted = 0

池级开关在「投资池维护」配置；本页只维护区间行。开关=0 时本表不参与校验。

关键规则

  • 页面只维护开放日配置，不改池树本身。
  • 同一池可配置多段历史/未来区间；调库只认「当日命中」的未删除区间。
  • 与 lock_flag / 冻结期独立：锁定优先硬拦；开放日是日期窗口约束。
