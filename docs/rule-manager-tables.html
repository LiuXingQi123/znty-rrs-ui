查询、校验与操作逻辑

按当前前后端实现归纳主要触发场景。

规则查询与详情

规则列表读取 rule_definition 并关联分类；详情继续加载 rule_param 与 rule_param_option。分类和预置选项分别由独立下拉接口读取。

新增与编辑

保存前校验规则名称、表达式和参数；主规则新增或更新后，对参数及参数选项采用删除旧集合、重新插入的全量替换方式，避免残留已移除配置。

启停与删除

editRuleStatus 只改变 active / disabled 状态；deleteRule 对 rule_definition 做逻辑删除。删除规则不会级联物理删除测试用例或既有运行历史。

QLExpress 执行

string 参数按字符串注入；number 参数先以 BigDecimal 精确解析，再根据是否含小数转换为 Double 或 Long。表达式异常显式返回失败并记录日志。

测试用例

用例主表保存名称、规则和最近结果，参数表保存每个变量的输入与期望。编辑用例时先更新主表，再全量替换参数。

运行与历史

单条或批量运行均调用相同规则执行逻辑；每次写 rule_test_run，并按执行步骤写 rule_test_run_log。删除用例不删除既有运行历史。

涉及表与读写场景

规则配置、测试用例和运行历史彼此关联，但生命周期并不完全级联。

关键规则与边界

规则保存、执行和测试运行的核心口径。

删除边界

规则采用逻辑删除；测试用例采用物理删除，但两者都不主动清除历史运行记录。

参数类型

当前明确支持 string 与 number；number 使用 BigDecimal 解析，避免直接以浮点字符串误判。

事务一致性

规则及参数、用例及参数、运行及日志分别在事务中维护，任一写入失败应整体回滚。

代码值口径

后端返回分类、状态等原始 code，中文由前端字典完成。执行引擎 QLExpress；number 用 BigDecimal；运行写入 t_rule_test_run / t_rule_test_run_log。
