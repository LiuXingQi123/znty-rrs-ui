迁移总览

建议按“代码路径 → 配置 → 数据库脚本 → 联调验证”的顺序推进。

推荐流程
1. 确认目标仓库明确前端静态页面、后端服务、SQL 脚本最终放在哪个公司项目路径。
2. 替换环境值替换 API 地址、数据库连接、库名、登录用户上下文、静态资源路径。
3. 执行建表脚本先 schema 后 demo，保持脚本顺序一致，确认 MySQL 8 默认字符集。
4. 启动联调后端启动后打开前端工作台，重点验证调库、审批、文档和脚本工具。
5. 回归清单检查菜单、接口、数据、附件、报告、字典和脚本工具是否都能正常使用。

公司 html / js / css 分离迁移方式

公司项目采用三文件分离结构时，只迁移 pages/ 里的功能页面；docs/ 文档页按需要单独处理。

公司前端结构

HTML：只复制 body 业务结构

先创建与本项目 pages 中同名的空壳 HTML，例如 my_matters.html。然后只复制原页面 <body> 内的业务 DOM，例如 <div id="my_matters">...</div>。

部署配置注意事项

迁移后最容易出问题的是跨域、静态资源、权限和脚本工具安全边界。

部署与权限

前端静态部署

如果使用 Nginx 或公司门户托管，确认 index.html、pages/、docs/、css/、js/ 都在同一相对路径下。

跨域与网关

本地前端默认请求 http://localhost:18090。迁移到网关后，优先改 js/api.js 的 baseURL，并确认 Cookie / Token / Header 策略。

脚本工具权限

脚本工具可以清表和执行 SQL 初始化。生产环境建议隐藏菜单，或只允许管理员在隔离环境使用。

迁移后验证清单

按下面顺序跑一遍，基本能覆盖前端、后端、数据库和文档页面的常见断点。

Checklist
