1. 选择 CRMW

分页查凭证（security_type=crmw），再选可绑定标的（排除 crmw）。

2. 加载上下文

并行读凭证/标的详情、ip_pool_status_crmw 当前池、可调 CRMW 池与历史申请。

3. 规则校验

强制 poolType=crmw；组合身份字段完整；通用/类型校验+联动互斥反阻断。

4. 创建申请

写 ip_adjust_log/step，批次号 CRMW 前缀，统一 submit_time。

5. 审批生效

仅 audit_status=20 写/软删 ip_pool_status_crmw；直通亦先锁池复核。

查询、校验与操作逻辑

与证券池调库同构；对象=「CRMW 凭证 + 标的证券」组合，落地表独立。

查询与选择

凭证列表 queryCrmwPage（仅 crmw）；可绑定证券 queryBindableSecurityPage（排除 crmw）。提交/日志同时携带 crmwScode、crmwName、crmwStype=crmw 与标的 securityCode。已移除市场代码 crmwMktcode，前后端不再传参。

池与权限

可调池仅 pool_type=crmw 且启用叶子；普通用户需 adjustable 权限。CRMW 在池读 ip_pool_status_crmw；标的普通池 ip_pool_status 仍参与来源池/观察池等规则。

调入 / 调出校验

通用：池锁定、品种/市场、pending（按 CRMW 组合+池组）、重复入/未入池、容量、来源/限制、互斥冲突、全局禁止、行业、开放日；弹性 soft_restrict 为警告不阻断。自动联动/互斥；互斥调出失败反阻断同组手工调入。特殊审批（互斥）默认排除信用债大库目标池。

提交与流转

直通：锁池动态复核后写/软删 ip_pool_status_crmw。非直通：audit_status=00 写步骤，仅最终 20 落池。前端调整说明失败时仅展示不通过原因；同组底色区分联动/互斥行。约 30s 防重复提交。

关键规则与边界

帮助区分当前状态、历史记录和流程实例。

关键差异

最终池状态只落 ip_pool_status_crmw；申请/步骤仍共享 ip_adjust_log、ip_adjust_step。批次号前缀 CRMW（证券/禁投为 BOND）。

对象约束

调库对象不能是 CRMW 凭证本身（标的必须非 crmw）；凭证与标的双身份字段必填。日志字段含 crmw_name/scode/stype，无 mktcode。

状态边界

仅 audit_status=20 改变 CRMW 池状态；00/11/21/99 等只改日志与步骤。三链路同构：校验/流转改动时需同步评估证券池与禁投池。
