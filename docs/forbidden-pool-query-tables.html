禁投池查询

查询风险类投资池（forbidden / observe / blacklist）的当前已生效状态，覆盖主体与证券，并按用户可查看权限过滤。

5 项表/表组2 个主要接口依据当前代码实现
流程总览禁投池分页查询涉及表规则说明文档目录

业务流程总览

从页面操作到数据库读写的主要路径。

1. 权限范围按 currentUserId 解析 viewable 可查看池 ID 集合。
2. 加载类型查询风险池中实际出现的证券类型选项。
3. 组合筛选按代码、类型、入池时间、调整人、发行人等分页。
4. 限定范围pool_type ∈ forbidden/observe/blacklist 且已生效。
5. 路径回填Service 将目标池名替换为完整层级路径。

禁投池分页查询

主数据来自已生效池状态；范围=三类风险池 ∩ 用户可查看池。

POST /api/v1/forbiddenPoolQuery/queryForbiddenPoolPage · querySecurityTypeList

查询主线

  • ip_pool_status 内连接未删除的 ip_investment_pool。
  • 固定 pool_type IN ('forbidden','observe','blacklist')，且 audit_status='20'、is_deleted=0。
  • Service 注入 viewablePoolIds（PermissionType.VIEWABLE）；空集合则无数据。
  • 左连 rrs_securityinfo 与字典；主体行再连 wind_cbondissuer 取发行人名称。
  • 排序：entry_time DESC, id DESC。

筛选条件

  • 证券代码/简称、证券类型、入池日期区间、调整人、发行人（主体名或债券发行人模糊）。
  • 存续/到期：仅对债券大类用 maturity_date 与当前日比较（active/matured）。
  • 类型下拉与列表共用同一风险池 + 权限范围。

涉及表、用途与触发场景

读写口径包含页面初始化、主业务接口及其正常后续流程。

表名 用途 读写 触发场景
ip_pool_status 风险池当前状态主数据 查询 分页、入池时间与调整人筛选
ip_investment_pool 限定三类风险池并提供池路径 查询 pool_type 过滤、名称与路径回填
ip_pool_permission 等 解析用户可查看池（VIEWABLE） 查询 viewablePoolIds 注入
rrs_securityinfo 债券发行人与日期字段 查询 资料展示、存续判断、发行人筛选
ais_inv_ods.wind_cbondissuer 主体名称（security_type=company） 查询 主体在池行发行人展示
dict_security_type 证券类型名称和大类 查询 列表展示、类型选项、存续过滤

关键规则与当前实现说明

  • 当前 SQL 固定 audit_status='20' 且 is_deleted=0，只展示已生效在池状态。
  • 范围是三类风险池类型，不是单一 forbidden，也不是写死池 ID 15/16/17（权限再裁剪可见子集）。
  • 可同时出现主体级与证券级（含主体同步自动入池的债券）记录。
  • 页面只读，不写业务表；无收藏能力（与证券池查询不同）。
